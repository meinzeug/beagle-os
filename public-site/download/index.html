Getting Started Architecture

Image formats

Choose the image that matches how you want to test, install, or mass-provision endpoints.

ISO

ISO installer

Use when you want to boot a machine or VM into the installer and deploy Beagle OS onto local storage.

  • Best for bare-metal rollout
  • Carries the installer flow
  • Pairs with VM-specific preset data
IMG

IMG direct flash

Use when you want a direct-flash artifact for repeatable endpoint media creation without a separate installer session.

  • Useful for appliance-style deployments
  • Fastest path to a prepared medium
  • Keeps the same runtime model
USB

USB live and installer payloads

Use the published payload, bootstrap, and helper scripts to create reproducible endpoint media or to rebuild sticks from automation.

  • Published with checksums
  • Supports live USB and installed thin clients
  • Feeds self-update and repair flows

Beagle host setup

Clone the repository and run the host installer on the machine that should become your Beagle control plane and virtualization host.

Core setup command

git clone https://github.com/meinzeug/beagle-os.git
cd beagle-os
./scripts/install-beagle-host.sh

Optional variables

INSTALL_DIR=/opt/beagle
BEAGLE_PUBLIC_UPDATE_BASE_URL=https://beagle-os.com/beagle-updates
PVE_DCV_PROXY_SERVER_NAME=srv.example.net
BEAGLE_SITE_PORT=443

VM-specific USB installer flow

Beagle keeps provisioning VM-centric. Public artifacts cover the generic media, and your own Beagle host generates the final VM-specific USB scripts.

  1. Install the Beagle control plane on your Beagle host.
  2. Prepare the target VM with Sunshine using the guest helper script.
  3. Open the VM in the Beagle WebUI and download the generated USB live or installer script for exactly that VM.
  4. Write the USB stick. The script downloads the host-served image payload and embeds the VM profile automatically.
  5. Boot the endpoint. Beagle OS comes up in runtime mode and connects to the assigned VM.

Checksum verification

Artifacts are published on beagle-os.com and can also be mirrored by your own Beagle host. Verify checksums before you distribute images.

cd ${INSTALL_DIR:-/opt/beagle}/dist
sha256sum beagle-os-installer-amd64.iso
sha256sum pve-thin-client-usb-bootstrap-latest.tar.gz

The public `SHA256SUMS` file and hosted status JSON are published together with the release so workflows and operators validate the same payload set.
