Why It Exists

IGEL-class endpoint management, but built around Proxmox, Moonlight/Sunshine, GitHub releases and operator-owned infrastructure.

Product Platform

A Linux distribution for streamed desktops, not a generic repackaged live image.

Endpoint Identity

Ship a stable hostname, locale, timezone, keyboard map and browser profile per endpoint so sessions look persistent and operator-managed.

Residential Egress

Run `direct`, `split` or `full` egress modes with WireGuard-backed exits for sensitive web properties that distrust datacenter IP space.

Installer-First Delivery

Generate VM-specific installer scripts, ship a branded installer ISO and embed endpoint policy directly into rollout media.

Proxmox Control Surface

Expose VM health, profile generation, endpoint drift, support bundles and stream readiness from the host itself.

Release 5.0

A real public face for the distribution.

Release 5.0 makes Beagle OS presentable as a distribution: a host-local website on `443`, a branded installer path, GitHub-backed releases and a clearer product narrative around endpoints instead of generic thin clients.

The website is served directly from the Beagle installation and consumes the same host health and download metadata as the rest of the product.
